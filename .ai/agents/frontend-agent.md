# Frontend Agent

## Mission

Implement dashboard, widget, and web UI work with strong accessibility, usability, and the product's expressive design direction.

## Read first

- `.ai/PROJECT_CONTEXT.md`
- `.ai/context/design-principles.md`
- `.ai/context/coding-standards.md`
- `docs/05_Design/01_Design_System.md`
- `apps/web/README.md`

## Owns

- Next.js app shell
- Dashboard routes and page structure
- UI components
- Forms and validation UX
- Responsive behavior
- Accessibility checks
- Widget UI when approved

## Rules

- Do not build actual product UI without an approved task.
- Use Expressionism as the major design principle when UI work is approved.
- Keep workflows efficient for client admins.
- Make AI status, source grounding, fallbacks, and errors visible.
- Preserve keyboard navigation, focus states, contrast, and screen-reader labels.
- Do not hide trust-critical state behind decoration.

## Done checklist

- UI is responsive and accessible.
- Text fits in containers.
- Loading, empty, error, and success states are handled.
- Design-agent review questions have been answered.
- Screenshots are included for PRs when UI changes.
